# Rappel exigences techniques de l'export

900x600@60 et hexcode :

- Largeur : 900px

- Hauteur : 600px

- Taux de rafraîchisement : 60 frames par secondes

- Mode colorimétrique durant la production :  rgba(rouge, vert, bleu et alpha) en 8bits (de 0 à 255 pour le rgb et de 0 à 255 pour le alpha)

- mode colorimétrique durant l'envoit de fichier : code hexadecimal (#RR-GG-BB-AA)

# devis GDD

Vous allez devoir reprendre l'image (TIMP-Pongx2\mettings\ébauche_finale_pong^2.jpg), et expliquez tout les mécanismes et les éléments du jeu.

## Remise

Pour la remise, je veux un fichier .zip venant d'un lien one drive ou google drive modifiable ou d'un fichier zip et tout ses éléments non-transférés dans un drive (GitHub ne compte pas). Voici les trois fichiers attendus dans le fichier zip :

- Un fichier Microsift 365 word ou un fichier AdobeIndesign ou un fichier affinity publish (une version modifiable si besoin)

- Un fichier pdf pour la version définitive (non-modifiable)

- un fichier texte (.txt) avec tout vos sources (s'il y a lieu) => nom de l'image ou du jeu : lien de la source (ex : jeu-vidéo Celeste : https://www.celestegame.com/)

- Langue : soit en français ou en anglais, les informations doivent être clairs et seulement dans la langue choisi (ex : GDD en anglais = pas phrase en français);

## éléments GDD

Voici les éléments pour le GDD : 

- Nom du jeu en gras centré;

- une ligne indiquant les créateurs du GDD centré après le titre du jeu (ex : document de conception de jeu crée par Marc-Alexandre Di Marco François);

- Un paragraphe en justifié sur les inspirations que aurez pu trouver (en liste avec ";" à la fin de chaque élément de la liste)

- Un paragraphe en justifié sur les choix artistiques (DA, assets, sons, etc.). Le but est de donner une identité **clair** afin que les programmeurs comprennent le sens artistique du jeu

- Deux paragraphes sur le déroulement du jeu en justifié. Un explique le déroulement d'une partie (UX) et l'autre explique les commandes et interfaces (UI). Cette partie doit être très bien détaillé, car les programmeurs vont s'en servir pour créer leir code selon votre vision.

- Écrire en gras à la droite (alignement), exactement ceci : "Le moteur de jeu utilisé sera Godot Engine 4.6."

- Aucune limite pour le nombre de pages.
